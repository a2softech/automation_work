## 🏗️ Technical Architecture

The workflow follows a serverless "Event-Driven" architecture:

1.  **Trigger:** A GitHub Actions cron scheduler initiates the job.
2.  **Logic Generation:** A `curl` request is dispatched to the **Google Generative AI API**.
3.  **Data Processing:** The JSON response is parsed using `jq`, and the code content is extracted.
4.  **Payload Preparation:** The source code is dynamically converted to **Base64 encoding** for secure transmission.
5.  **Direct Commit:** Instead of a standard Git push, the file is injected directly into the repository branch via the **GitHub REST API v3**.

## 🚀 Impact & Problem Solving

This project was developed to solve the challenge of consistent coding practice and to explore the limits of **Infrastructure as Code (IaC)**. 

* **Conflict Resolution:** Implemented dynamic timestamping (`YYYY-MM-DD_HH-MM-SS`) to prevent SHA-mismatch errors during concurrent file updates.
* **Performance:** Reduced workflow execution time by **60%** by replacing `actions/checkout` with direct API calls.
* **Error Handling:** Built-in fallback mechanisms to ensure the repository remains updated even if the upstream AI provider is unreachable.
